Detailed Description

Network scale sedimentary system object.

Contains all informations regarding the sedimentary system, i.e. its initialisation and its dynamicity through the simulation.

Globaly contains:

  • The reach initial morphological informations
  • The sediment transfer functions
  • The sediment storing matrices
Parameters
reach_dataThe reach data inputs (ReachData object)
networkNetwork structure (graph)
timescaleTime step number (int)
ts_lengthTime step length (float)
save_dep_layerOption for saving deposit layer
psiSediment size class vector in psi scale (1d array)
phiPorosity (default: 0.4)
minvelminimum velocity (default: 0.0000001)
n_metadatanumber of metadata column (default: 1, for storing initial provenance)

Compute the mobilised volumes in reach n at time step t.

This function compute the mobilised volumes from a reach, considering the eventual continuig cascades (cascade coming from upstream but not stopped by time). It can mobilised a volume from the reach (Vmob) and/or deposit some of the continuing cascade (that are now stopped by energy).

Note
The order in which we select the continuing cascade to be deposited is from largest times (arriving later) to shortest times (arriving first). Hypotheticaly, cascade arriving first are passing in priority, in turn, cascades arriving later are deposited in priority.
Parameters
VdepReach deposit layer
tr_cap_per_sTransport capacity
passing_cascadesList of cascade (the ones that did not end the time step in this reach = continuing cascades)
Returns
Vmob New volume mobilised from the reach deposit layer
passing_cascades Liste of cascades (continuing cascade that actually manage to pass the outlet)
Vdep_new Updated reach deposit layer

Select uppermost layers available in a reach.

This function searches uppermost layers from a volume of layers, to correspond to a maximum volume (V_lim). Passing cascade can be integrated to the top of the volume (Qpass_volume). The maximum volume can represent for example the active layer, i.e. what we consider as active during the transport process, or a maximum to be eroded per time step.

Compute the velocity of one sediment volume.

Parameters
volumeone sediment volume
Q_reachReach water discharge [m3/s] at this time step
vReach water velocity [m/s]
hReach water height
tTime step
nReach index
indx_vel_partitionIndex for velocity calculation method among the different size classes
indx_tr_capIndex for transport capacity formula
indx_tr_partitionIndex for transport capacity paritionning
Returns
the computed velocities (1d array, size: n_classes)
Note
The transport capacity [m3/s] is calculated on this volume, and the velocity is calculated by dividing the transport capacity by a section (hVel x width x (1 - porosity)).
For partionning the section among the different sediment size class in the volume, two methods are proposed. The first one put the same velocity to all classes. The second divides the section equally among the classes with non-zero transport capacity, so the velocity stays proportional to the transport capacity of that class.
